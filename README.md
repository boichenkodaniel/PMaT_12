### Запуск через Docker Compose (рекомендуется)

Поднимает два сервиса: FastAPI-приложение и PostgreSQL.

```bash
docker-compose up --build
```

Приложение будет доступно по адресу: http://localhost:8000  
Интерактивная документация (Swagger UI): http://localhost:8000/docs

#### Переменные окружения

| Переменная | Обязательная | Значение по умолчанию | Описание |
|------------|-------------|----------------------|----------|
| `DATABASE_URL` | ✅ | — | URL подключения к PostgreSQL. Пример: `postgresql+psycopg://user:password@host:5432/dbname` |
| `ALLOWED_ORIGINS` | ❌ | `http://localhost:3000,http://127.0.0.1:3000` | Список разрешённых CORS-источников через запятую |

```bash

```

### Задание 1 — REST API для управления книгами и авторами

Приложение предоставляет CRUD-операции для управления книгами и авторами в библиотеке. Реализована нормализованная схема БД: таблицы `authors` и `books` со связью **один-ко-многим** через внешний ключ `author_id`.

#### Схема базы данных

```
┌─────────────────┐         ┌──────────────────┐
│    authors      │         │      books       │
├─────────────────┤         ├──────────────────┤
│ id (PK)         │◄────────┤ id (PK)          │
│ name (UNIQUE)   │   1:N   │ title            │
│ bio             │         │ author_id (FK)   │
│ created_at      │         │ isbn (UNIQUE)    │
└─────────────────┘         │ published_year   │
                            │ is_available     │
                            └──────────────────┘
```

**Индексы:**
- `ix_books_id` — на первичный ключ книг
- `ix_books_title` — ускорение поиска по названию
- `ix_books_author_id` — ускорение выборки по автору
- `ix_authors_id` — на первичный ключ авторов

**Ограничения:**
- `UNIQUE` на `authors.name`
- `UNIQUE` на `books.isbn`
- `ForeignKey` `books.author_id → authors.id` с `ON DELETE RESTRICT`

#### Доступные эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Корневой эндпоинт (информация об API) |
| GET | `/authors` | Получить список всех авторов |
| GET | `/authors/{author_id}` | Получить автора по ID |
| POST | `/authors` | Создать нового автора |
| PUT | `/authors/{author_id}` | Обновить автора |
| DELETE | `/authors/{author_id}` | Удалить автора |
| GET | `/books` | Получить список всех книг |
| GET | `/books/{book_id}` | Получить книгу по ID |
| POST | `/books` | Создать новую книгу |
| PUT | `/books/{book_id}` | Обновить книгу (полностью или частично) |
| DELETE | `/books/{book_id}` | Удалить книгу |

#### Модели данных

**Author:**

| Поле | Тип | Ограничения | Обязательное |
|------|-----|-------------|--------------|
| `id` | integer | Автоинкремент | ✅ (только в ответе) |
| `name` | string | 1–200 символов, уникальное | ✅ |
| `bio` | string | — | ❌ |
| `created_at` | datetime | Автоматически `now()` | ✅ (только в ответе) |

**Book:**

| Поле | Тип | Ограничения | Обязательное |
|------|-----|-------------|--------------|
| `id` | integer | Автоинкремент | ✅ (только в ответе) |
| `title` | string | 1–200 символов | ✅ |
| `author_id` | integer | `≥ 1`, ссылка на `authors.id` | ✅ |
| `isbn` | string | 1–20 символов, только цифры и дефисы, итого 10 или 13 цифр, уникальное | ✅ |
| `published_year` | integer | 1450 – текущий год | ✅ |
| `is_available` | boolean | По умолчанию `true` | ✅ |

#### Примеры запросов и ответов

#### Валидация данных

Pydantic-модели автоматически проверяют входные данные:

**Author:**
- `name` — строка от 1 до 200 символов, уникальная
- `bio` — строка (опционально)

**Book:**
- `title` — строка от 1 до 200 символов
- `author_id` — целое число ≥ 1 (ссылка на существующего автора)
- `isbn` — строка от 1 до 20 символов, содержит только цифры и дефисы, итого 10 или 13 цифр, уникальная
- `published_year` — целое число от 1450 до текущего года
- `is_available` — булево значение (по умолчанию `true`)

Пример ошибки валидации:
```bash
curl -X POST "http://localhost:8000/books" \
  -H "Content-Type: application/json" \
  -d '{"title": "", "author_id": 1, "isbn": "978-0-123456-78-9", "published_year": 2020}'
```
Ответ: `422 Unprocessable Entity` с описанием ошибок.

#### Безопасность

Реализованы меры защиты на уровне приложения:

| Мера | Реализация |
|------|-----------|
| **XSS-санитизация** | `html.escape()` в валидаторах `name`, `bio`, `title` — экранирование HTML-тегов |
| **CORS** | Whitelist из `ALLOWED_ORIGINS`; разрешены только `GET`, `POST`, `PUT`, `DELETE` |
| **Rate limiting** | `slowapi`: 5/мин для `POST /authors`, 10/мин для `POST /books`, 30/мин для чтения; отключается в `TESTING=true` |
| **Security-заголовки** | `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `Strict-Transport-Security`, `Referrer-Policy`; `CSP` отключена для `/docs` и `/redoc` |
| **Нет хардкода паролей** | `DATABASE_URL` обязательна из переменных окружения |
| **Логирование** | Security-события (создание/удаление, ошибки) записываются через `logging.getLogger("security")` без PII |
| **Обработка исключений** | Глобальные `exception_handler` для `NotFoundError` (404), `ConflictError` (409), `Exception` (500) |
| **Управление сессиями БД** | `@contextmanager _db_session()` — гарантированный `rollback` и `close` |

Подробный аудит безопасности приведён в файле `Task7_Report.md`.

### Рефакторинг модуля расчёта штрафов

Исходная реализация представляла собой монолитную функцию `calc(data)`, принимающую кортеж с неинформативными индексами и содержащую дублирующуюся логику расчёта для каждого типа книги.

| Проблема | Исправление | Зачем |
|----------|-------------|-------|
| Магические числа | Вынесены константы (`WEEK_THRESHOLD_DAYS`, `MONTH_THRESHOLD_DAYS`) и тарифы в словари (`DAILY_RATE`, `WEEKLY_SURCHARGE`, `MONTHLY_SURCHARGE`) | Изменение тарифа в одном месте, а не в дублях кода |
| Неинформативные аргументы | Создан `PenaltyInput` (`@dataclass`) с именованными полями (`overdue_days`, `book_type`) | Читаемость и валидация на уровне типов |
| «Сырые» строки типов | `BookType(str, Enum)` | Типобезопасность, автодополнение IDE, единый источник истины |
| Дублирование логики | Единая функция `calculate_penalty` без повторения кода для каждого типа | Переиспользование, проще тестировать |
| `print` в бизнес-логике | Убраны все `print`, логирование вынесено за пределы модуля | Чистая бизнес-логика без сайд-эффектов |
| Ручная работа с файлом | Убрана запись в файл из модуля расчёта | Разделение ответственности: расчёт ≠ логирование |
| Отсутствие валидации | `ValueError` при `overdue_days < 0` и при неизвестном `book_type` | Раннее обнаружение ошибок, предсказуемое поведение |
| Нет type hints | Добавлены аннотации ко всем функциям и константам | Статический анализ, самодокументируемость |
| Неясное имя `calc` | Переименовано в `calculate_penalty` + обёртка `calculate_penalty_from_raw` | Читаемость и обратная совместимость |
